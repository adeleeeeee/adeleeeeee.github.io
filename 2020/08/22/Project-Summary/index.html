Project Summary

2020-08-22

Project Summary

As the development of technology growing rapidly, e-commerce became extremely popular in recent years. Therefore, businesses would not be restricted to only sail products offline but have a broader market. However, bigger market means more competitions between corporations, especially for e-commerce, which costumers could have a clearer and easier comparison between similar products, and business has not developed a mature online marketing plan yet. How to recommend suitable products to customers and how to attract more potential costumers became crucial questions for business. Under this situation, they need a more unbiased and accurate scheme for marketing. Reviewing costumer’s data seem to be a valuable way of costumer intention prediction. For example, the biggest online shopping application in China—Taobao would recommend similar products according to browsing history.

My research question is using machine learning for modeling real-time online shopping intention prediction, and it would use two class classification machine learning. The dataset includes the number of times that costumer would click to a specific page, the duration of staying on a specific page, the bounce rate (exit rate), the revenue which indicates the transaction of visitors and the channel that costumers used. This information in the dataset could reveal the performances of different costumers of different areas in online stores. The objective of this research is to provide a more useful suggestion for marketing decisions through application of machine learning, which could transfer big data to a more visualize and convenient model.

The dataset we received in the start only contains a bunch of numbers, but by processing further exploration and visualization, the implicit information hide in the dataset could be revealed by a more vivid and clear way. Moreover, data visualization could help to demonstrate the correlation between variables that would affect costumer’s final decision. Therefore, data exploration and visualization are crucial steps in data processing and beneficial to comprehension.

What is more, in all of the variable，page values have the most positive correlation with revenue, which infers that business should attach more importance on this variable.

The dataset we used in data processing includes lots of unnecessary variables, which may lower the accuracy of the final conclusion of costumer intension. Feature engineering is used to help me finding the most relevant variables. In this case, I could use feature engineering to remove variables that have the lowest correlation with revenue. After finishing this step, I could utilize a more useful training data for machine learning. Hence, it is an important step in machine learning and could help to obtain a better conclusion.

There are total three steps in feature engineering：standardize features, encode categorical features and feature selection. First, standardizing features by making the features’ mean and unit variance more equal, which could make the features more comparable and conclude a better result. Moreover, it also could make sure that important features would not be treating with bias. Then, encoding categorical features though label encoding and one-hot encoding. This is used to make data understandable to machine learning algorithms and decrease the unnecessary columns of same features. Finally feature selection would separate into two small steps: processing correlation graph to measure the strength of relationship between other variables with revenue and Using Recursive feature elimination (RFECV) to rank the feature by repeatedly choosing the best or worst performing features with each time. This process could help to eliminate the low rank features. However, it has some weakness that the optimization may not be always accurate because of relationship between features.

After finishing these steps above, the selected features are, ‘Administrative', 'Informational', 'Product Related', 'Bounce Rates', 'Exit Rates', 'Page Values', 'Special Day', 'Month', 'Operating Systems', 'Browser', 'Region', 'Traffic Type', 'Visitor Type' and 'Weekend'.

In this project, I choose two machine learning models, logistic regression algorithm and k-nearest neighbors’ algorithm (KNN) to run the selected features. The precondition is that all the data should be numerical and in normal distribution. Logistic regression is a type of algorithm that would only give yes or no answers, which is very suitable for binary classification problems. It works by first using linear regression to conclude a correlation, then the logistic function would set a mean to be threshold, which turns correlation into binary form. Under this circumstance, the threshold is 0.5. Therefore, if the feature is bigger than 0.5 the prediction would be 1, otherwise would be 0. In KNN, it includes k training data in feature space. The output depends on the result of the ‘vote’, which means the class that have more votes in a certain number of nearest neighbors would be the final conclusion. In this case, I set n-neighbors to be five, which means the nearest five training data, weights to be ‘uniform’, and using Euclidean distance, which is when p equals two. Euclidean distance represents the straight-line distance, which is also the nearest distance between two points, like the hypotenuse of right triangle. The limitation is that I only utilize two machine models, which may lower the accuracy of the prediction.

After running the models, I used cross validation to evaluate the performances of the models, with training data only. It separates training data into k-folders. Then, in each folder, it will take one group of data to be ‘validation data’ and others be ‘training data’. This is a technique that could make prediction more accurate and could prevent overfitting, which is when the prediction is fitting to well with the training data and would likely have a lower accuracy in test data. For logistic regression, the validation score is 0.884 and the prediction score is 0.886, which are very close. For KNN, the validation score is 0.880 and the prediction score is 0.882. For, logistic regression, accuracy is 0.887, precision is 0.759, recall is 0.395 and F1 score is 0.520. For KNN, accuracy is 0.882, precision is 0.712, recall is 0.401 and F1 score is 0.513. As we can see, logistic regression has a higher accuracy and F1 score then KNN. Then, I changed some parameters in KNN and logistic regression to optimize the prediction.

Conclusion

In conclusion, I used machine leaning models to predict real-time online shopping intention. I selected features of the model by recursive feature elimination (RFECV). Moreover, I checked several machine learning models by cross validation technique and confusion matrix. Finally, I decided to use logistic regression. The result of the model is 0.886. Changing of parameters of models and using of more models could help to make further improves of the results.
